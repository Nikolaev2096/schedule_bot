## Краткое описание 

Бот предназначен для помощи студентам разных учебных заведений автоматизировать процесс получения расписания. 

Проект поддерживает работу с получением данных из учебного расписания, расписания сессии.
Бот имеет возможность автоматической рассылки уведомлениий по времени, которое указывает каждый пользователь и разграничение доступа за счет системы с ролями пользователей (студенты, преподаватели, абитуриенты, администраторы). 

## Что умеет бот

- [x] Создавать записи пользователей в БД 
- [x] Обрабатывать ошибки ввода пользователей 
- [x] Выдавать учебное расписание в разных срезах (по неделям, группам, преподавателям, дням недели, календарным дням и кабинетам)
- [x] Выдавать расписание сессии 
- [x] Отправлять уведомления со звуком и без звука разным группам пользователей. Под группами пользователей подразумевается какая-либо конкретная группа студентов, все студенты, преподавательский состав, администраторы.
- [x] Автоматически рассылать расписание на следующий день
- [x] Поддерживать различные роли пользователей (студент, преподаватель, абитуриент, администратор)
- [x] Загружать расписание из документов Word (.docx)
- [x] Работать с двухнедельным расписанием (четная/нечетная неделя) с возможностью адаптации под 3-х и 4-х недельное расписание 
м другие функции

## Технологии

Проект построен на следующих технологиях и библиотеках:

- **aiogram 3** - современный асинхронный фреймворк для создания Telegram ботов
- **SQLAlchemy 2** - ORM для работы с базой данных
- **aiosqlite** - асинхронный драйвер для SQLite
- **APScheduler** - планировщик задач для автоматической рассылки уведомлений
- **python-docx** - библиотека для парсинга документов Word (.docx)
- **Poetry** - менеджер зависимостей и виртуального окружения

## Описание модулей

### `handlers/` - Обработчики команд и сообщений
Модули, отвечающие за обработку пользовательских команд и взаимодействие с ботом:
- **base_handlers.py** - базовые обработчики: регистрация пользователей, главное меню, смена группы/роли
- **schedule_handlers.py** - обработчики для получения расписания 
- **abitur_handlers.py** - обработчики для абитуриентов
- **admin_handlers.py** - административные функции: загрузка расписания, статистика, рассылки, управление администраторами

### `database/` - Работа с базой данных
Модули для работы с SQLite базой данных:
- **model.py** - SQLAlchemy модели: Users, Schedule, Session, ScheduleImproves, HomeWork, HomeWorkFiles. База не 3nf.
- **user_func.py** - функции для работы с пользователями (получение данных, проверка прав, ролей)
- **schedule_func.py** - функции для работы с расписанием (получение по группам, преподавателям, дням, неделям)
- **misc.py** - вспомогательные функции для работы с данными расписания
- **midleware.py** - middleware для автоматического управления сессиями БД

### `transformation/` - Преобразование данных
Модули для парсинга и преобразования расписания из документов Word:
- **transform_education_schedule.py** - преобразование учебного расписания из .docx в структурированные данные БД
- **transform_session_schedule.py** - преобразование расписания сессии из .docx в структурированные данные БД

### `addons/` - Дополнительные функции
Вспомогательные модули:
- **notifications.py** - система уведомлений: автоматическая рассылка расписания на следующий день, групповые и массовые уведомления
- **weeks.py** - функции для работы с учебными неделями (определение текущей недели, полугодия)

### `templates/` - Шаблоны форматирования
Модули для форматирования вывода:
- **schedule_template.py** - шаблоны форматирования учебного расписания
- **session_template.py** - шаблоны форматирования расписания сессии
- **txt_templates.py** - текстовые шаблоны сообщений бота

### `keyboards/` - Клавиатуры
Модули для создания интерфейса бота:
- **inline_keyboards.py** - inline-клавиатуры (кнопки под сообщениями)
- **reply_keyboards.py** - reply-клавиатуры (клавиатуры внизу экрана)

### `states/` - Состояния FSM
- **states.py** - endpoint'ы машины состояний (FSM) для обработки диалогов с пользователем

### `main.py` - Точка входа
Инициализация бота, подключение роутеров, настройка планировщика задач и запуск polling

### `examples` - Шаблоны расписания 
Примерные шаблоны разметки расписания

### `config.json` - состояние сдвигов и полугодий 
Структура конфигурации: 
```json
{
    "week_inverse": 1,
    "half_year_num": 0
}
```
Конфигурация создается автоматически после запуска системы 

### `documents/` - Временное хранилище для файлов расписания 

При инициализации обновления расписания - в эту директорию загружаются файлы расписания, после обработки они удаляются

## Использование 

### Загрузка
```bash
git clone https://github.com/Nikolaev2096/schedule_bot/
````
> [!IMPORTANT]
> Перед сборкой и запуском необходимо создать файл конфигурации .env

`.env`  должен содержать следующие значения:
- `TOKEN` - токен вашего Telegram бота
- `admin_ID` - ваш Telegram ID

синтаксис написания: 
```text
TOKEN = токен_бота
admin_ID = ващ_телеграйм_айди
```


Также необходимо добавить файл `templates/txt_templates.py` с текстовыми шаблонами, образец находится в `/templates/txt_templates_example.py`

#### Docker:

> [!NOTE]
> Рекомендуется использовать docker или docker compose 

```bash
docker build -t schedule-bot .
docker run schedule-bot -d
```

#### Docker-compose 
```bash
docker compose build 
docker compose ud -d
```

### Без использования Docker 

#### Linux/macOS:
```bash
python3 -m venv venv 
source venv/bin/activate 
pip3 install poetry 
poetry install 
python3 main.py 
```

#### Windows:
```shell
python -m venv venv 
venv\Scripts\activate 
pip install poetry 
poetry install 
python main.py 
```
